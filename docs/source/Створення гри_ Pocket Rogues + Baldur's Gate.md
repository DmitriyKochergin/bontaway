## **2\. Брейншторм механік: Перші цеглинки специфікації**

### **А. Розумний Віртуальний Джойстик (Рішення проблеми PR)**

Проблема Pocket Rogues: жорстко фіксований центр джойстика. Якщо палець промахнувся повз центр, персонаж стоїть, а гравець втрачає секунди і вмирає.

* **Рішення (Динамічний джойстик):** Центр джойстика з'являється **там, де гравець торкнувся екрана** (у лівій половині). Точка першого дотику стає координатами (*x*0​,*y*0​). Куди б палець не рушив далі — від цієї точки рахується вектор напрямку і швидкості. Якщо палець відірвали — джойстик зникає. Це повністю прибирає «сліпі натискання».

### **Б. RTwP на Тайловій Мапі**

Оскільки мапа складається з тайлів (сітки), виникає питання: як реалізувати реалтайм-рух і тактичну паузу?

* **Концепція «Тіків» (Ticks):** Гра працює в реальному часі, але внутрішньо розбита на мікро-раунди. Коли ти затискаєш джойстик, персонаж плавно переходить між тайлами.  
* **Пауза за триггером:** Пауза вмикається автоматично (опціонально), коли ворог потрапляє в лінію зору (True Fog of War відкриває ворога \-\> гра миттєво зупиняється, екран злегка темніє / знебарвлюється).  
* **Каст заклинань:** Натиснув на кам'яну кнопку заклинання \-\> гра стала на паузу \-\> вибрав ворога або область \-\> зняв з паузи \-\> персонаж почав читати заклинання (*«Vita, Mortis...»*), ворог у цей час біжить до нього. Якщо ворог встиг вдарити і збити концентрацію — каст скасовано.

### **В. Дослідження без «дитячих садків» (No Manuals)**

Якщо немає туторіалів, світ має говорити сам за себе через екологічний дизайн (Environmental Storytelling):

* **Експериментальна магія:** Гравець знаходить сувій із невідомим заклинанням. Опису немає. Тільки назва або руна. Що воно робить? Потрібно кастанути і подивитися. Можливо, воно підпалить ворога, а можливо — викличе вибух навколо кастера.  
* **Інтерактивне оточення:** Замість напису «натисніть ричаг, щоб відкрити двері», гравець бачить кабель або ланцюг, що йде від важеля під стіну. Натиснув — почув важкий звук каменю, двері піднялися.

### **Г. Інтерфейс та Аудіо-вайб**

* **Кам'яний UI:** Кнопки інтерфейсу мають виглядати як важкі плити з барельєфами рун. При натисканні — анімація вдавлювання всередину з глухим, низькочастотним звуком тертя граніту або удару каменю об камінь. Жодних «пластикових» чи неонових кліків.  
* **Аудіо-черга:** Звук касту заклинання має пріоритет. Якщо грає ембіент підземелля, у момент вимови латинської фрази касту, інші звуки мають злегка приглушуватись (ефект дакінгу), щоб голос звучав моторошно і вагомо.

### **Відповідь на питання 4: Чому Vampire Survivors пішли з Phaser?**

*Vampire Survivors* дійсно починався як браузерна гра на **Phaser 3**. Автор (poncle) написав її за пару місяців "на коліні". Гра стала суперхітом, і коли онлайн зріс, він зіткнувся з двома проблемами HTML5-рушіїв:

1. **Специфіка Garbage Collection (збір сміття) в JS:** Коли на екрані одночасно *15 000 ворогів*, кожен з яких постійно створює об'єкти (досвід, монети, снаряди), рушій JavaScript в браузері починає на мікросекунди "замерзати", щоб очистити пам'ять. Це викликало мікрофризи.  
2. **Портування на консолі (Xbox, Nintendo Switch):** Запустити HTML5/JS гру на Switch чи Xbox напряму дуже важко, перформанс там жахливий через слабкі процесори мобільних архітектур консолей.

**На що перейшли?** Гра була повністю переписана на **Unity**. **Чи критично це для нашої гри?** **Ні, абсолютно.** У нас тактична RPG (RTwP) з обмеженою кількістю ворогів у кімнатах (як ти написав: групи бандитів, зграї тварин). У нас на екрані не буде 5000 юнітів одночасно. Phaser ідеально впорається з геометрією, світлом та десятком-двома активних ворогів на 60 FPS навіть на слабкому смартфоні.

## **Аналіз та розвиток твоїх ідей: Брейншторм механік**

### **1\. Розумний джойстик, Стелс та Шум (Синхронізація пунктів 1 та 1.1)**

Ідея з вектором швидкості (чутливістю натискання) ідеально лягає на стелс-механіку.

* **Механіка "Радіус Шуму":** Навколо персонажа є прихований (або ледь помітний візуальний) радіус шуму.  
  * Злегка відхилив джойстик — персонаж йде навшпиньки. Шум мінімальний.  
  * Вдавив джойстик на максимум — біжить, шум на всю кімнату.  
  * Важка броня (лати) збільшує базовий радіус шуму, шкіряна куртка — зменшує.  
* **Зір ворогів (Конус зору):** Вороги мають кут огляду (наприклад, 120 градусів перед собою). Якщо ти підкрадаєшся ззаду (зі спини) — вони тебе не бачать. Але якщо ти біжиш і сильно шумиш — вони почують шум спиною, розвернуться і увімкнеться автопауза.

### **2\. Динамічне світло та True Fog of War (Пункт 1.1 \+ 10\)**

* **Візуал ліхтаря:** Персонаж у підземеллі тримає смолоскип/ліхтар або магічне світло. Промені світла прораховуються в реальному часі (в Phaser це робиться через рейкастінг на тайлах). Стіни кидають реалістичні довгі тіні.  
* **Тактична темрява:** Оскільки за стіною нічого немає (повна темрява), гравець бачить лише те, куди падає світло. Ворог може стояти за кутом у тіні. Ти робиш крок, промінь світла падає на гобліна — мапа миттєво сіріє, гра влітає в паузу. Ефект несподіванки 10/10.

### **3\. Магія в стилі BG2: Спасброски та Снаряди (Пункти 2, 5, 6\)**

* **Візуалізація Спасброску (Saving Throw):** Коли ти кастуєш "Страх" по області, вороги, які потрапили в радіус, кидають внутрішній кубик (наприклад, проти магії). Над головою ворога, який встояв, вилітає чіткий кам'яний текст: **\[СПАСБРОСОК: УСПІХ\]**, а над тими, хто злякався — іконка жаху і вони починають бігти геть. Це дає чітке розуміння: магія спрацювала, механіка чесна, просто ворог виявився сильним/вдачливим.  
* **Прив'язка до цілі (Homing Spells):** "Перст смерті" чи "Лікування" вилітають як снаряди і летять за ворогом, навіть якщо той тікає за кут. Але якщо ворог встиг закрити за собою двері (або двері зачинилися) — снаряд вдаряється в двері. Це чесно.

### **4\. Чесний світ: Ніякого сміття в бочках (Пункти 8, 12, 13\)**

* **Екологія підземелля:** Бочки — це просто бочки з водою або вином для бандитів. Якщо тобі потрібні гроші чи артефакти — шукай лігво.  
* **Унікальність ворогів:** Ніякої лінійної прогресії рівнів ("Гоблін 1 рівня", "Гоблін 80 рівня"). Якщо це Гоблін-нальотчик — у нього фіксовані HP (наприклад, 12\) і удар на 1d6. Якщо ти зустрів у глибокій печері істоту з назвою "Гоблін-Варлок" — він виглядає інакше (ритуальна маска, посох) і кастує вогняні кулі на 3d6. Ти відразу розумієш рівень загрози по візуалу.  
* **Високорівневі зони (High-risk, High-reward):** Ти можеш на першому рівні випадково знайти замасковані залізні двері в лігво Вампіра. Якщо ти зайдеш — він вб'є тебе за один хід. Але якщо ти проявиш архі-мозок: викличеш призовів, обкастуєшся захистами, закидаєш його сувоями здалеку через тактичну паузу і чудом переможеш — ти забереш його легендарний плащ і станеш напівбогом на початкових етапах гри. Це найкраще відчуття в RPG.

## **1\. Phaser 3 проти Unity: Чи є обмеження? (Пункт 4\)**

Давай чесно розберемо плюси і мінуси обох рушіїв у розрізі твого проекту (2D тайли, світло, WebView).

### **Unity**

* **Плюси:** Величезна екосистема, нативна комбіляція під мобілки (Android/iOS) без WebView, легке керування аудіо-потоками, готові інструменти для оптимізації великої кількості 2D-спрайтів (Tilemap, Universal Render Pipeline для світла).  
* **Мінуси:** **Важкий для браузера.** WebGL-білд Unity важить мінімум 15-30 МБ (навіть пустий), довго завантажується на мобільних браузерах і часто вилітає через нестачу оперативної пам'яті в Safari/Chrome на смартфонах.

### **Phaser 3**

* **Плюси:** Ідеальний для браузера. Гра запускається миттєво (вага ядра рушія \~1 МБ). На WebGL працює реактивно. Повністю контролюється через TypeScript.  
* **Мінуси:** Потребує сторонніх обгорток (Capacitor) для мобілок. Нативні функції (наприклад, складне аудіо без затримок) треба підключати через плагіни.

### **Чи є в Pocket Rogues щось, що не можна повторити на Phaser 3?**

**Ні. Абсолютно нічого.** Pocket Rogues — це класична 2D-гра з ортогональною/ізометричною сіткою, анімацією спрайтів та розрахунком колізій. Phaser 3 створений саме для цього. Динамічні тіні від смолоскипа, рух ворогів, «кам'яний» UI — все це на Phaser працюватиме навіть швидше і плавніше в браузері, ніж Unity-веб-білд. Головне — писати чистий код і не створювати сміття в пам'яті (Garbage Collection), про що ми говорили раніше.

## **2\. Механіка «Дофамінового детоксу» (Година на день) (Пункт 2\)**

Ідея обмежити час гри, щоб гравець не «перегорів» за ніч, а думав про гру протягом дня — це сміливий психологічний дизайн. Зараз ігри навпаки намагаються висмоктати 100% часу. Твій підхід виділить проект серед усього ринку.

### **Плюси (Pros):**

* **Ефект дефіциту:** Те, що обмежено, стає ціннішим. Гравець чекатиме наступного дня, як свята.  
* **Простір для мислення:** Коли гра вимикається на цікавому місці (наприклад, перед кімнатою зі статуями SCP), гравець весь день на роботі чи навчанні прокручує в голові: *«Так, статуї ходять у темряві... Як мені їх обійти? Може викликом світляка?»*. Це і є «вайб кодинг» для мозку.  
* **Ніякого "гринда":** Гра не перетворюється на другу роботу.

### **Мінуси (Cons):**

* **Роздратування:** Якщо гравця викидає з гри посеред напруженого бою, це викличе агресію, а не детокс.  
* **Сейв-скам часу:** Гравці на мобилках намагатимуться переводити системний час назад, щоб обдурити таймер.

### **Моя версія реалізації (The Autonomous Solution):**

Замість тупого таймера («Твій час вийшов, пока\!») або мобільних штампів з «енергією», ми вводимо **ЛОРНУ (ігрову) концепцію Втоми або Психічного виснаження**.

* **Механіка «Ментальний Саван»:** Підземелля токсичне для розуму. Що хвилини твій «Радіус Огляду» (True Fog of War) стає трохи меншим, а шепіт у темряві (звуковий ембієнт) — гучнішим. Кольори гри тьмяніють. Через годину реального часу твій екран стає майже повністю чорним, видно лише 1 тайл навколо персонажа, а в логу пише: *«Твій розум застеляє саван. Ти більше не розрізняєш стіни. Тобі потрібен відпочинок у таборі»*.  
* Гравець сам змушений вийти, бо грати стає фізично неможливо і небезпечно — у такій темряві статуї SCP вб'ють його за секунду. Гра зберігається, і щоб «відновити розум», реальний час має пройти до наступної доби. Чесно, атмосферно, без штучних банерів.

## **3\. Прототип: Кільце-Паразит та Смуга Перешкод (Пункт 3\)**

* **Перешкода 2 (Зигзаг):** Кільце починає «пульсувати» (нарощувати швидкість). Тепер джойстик не просто інверсований, його чутливість змінюється кожні 2 секунди (вектор швидкості плаває). Кільце буквально «котить» тебе до краю прірви.

Персонаж котиться вперед і впирається в дупу власного призваного вовка\! Вовк працює як живий бампер, не даючи персонажу впасти в прірву. ### **Ритуал безпечного відпочинку**

Ти правий, просто сісти посеред коридору і чекати відновлення не можна — з'їдять дикі тварини чи патруль бандитів.

* **Сигнальна магія (Вайб BG2):** Гравець має знайти глухий куток, закрити двері (якщо вони є) і кастанути на вхід у кімнату заклинання **«Сигнальна Руна» (Alarm Spell)** або «Магічна пастка».  
* **Геймплей відпочинку:** Гравець натискає кнопку «Відпочинок». Екран повністю гасне. Якщо кімната була безпечною — ментальне виснаження скидається. У тебе 1 HP, нуль витривалості і секунда, щоб придумати план порятунку.

### **Захист від обману часу**

Оскільки ми робимо веб\-гру/WebView, перевірка часу через нативний пристрій легко обходиться. Найкраще рішення — **простий запит на безкоштовний серверний API точного часу** (наприклад, WorldTimeAPI) у момент запуску гри. ## **3\. Наш перший крок у коді (Прототипування джойстика)**

Оскільки для вайб\-кодингу на Phaser 3 ідеально підходить TypeScript, давай створимо серце нашого майбутнього прототипу — **Розумний віртуальний джойстик із розрахунком швидкості та шуму**.

Ось як виглядатиме базова логіка цього компонента в Phaser 3:

import Phaser from 'phaser';

export class AdvancedJoystick {  
    private scene: Phaser.Scene;  
    private startPoint: Phaser.Math.Vector2 | null \= null;  
    private currentPoint: Phaser.Math.Vector2 | null \= null;  
      
    // Налаштування механіки  
    private maxDistance: number \= 100; // Радіус джойстика в пікселях  
      
    // Вихідні дані для персонажа  
    public moveVector: Phaser.Math.Vector2 \= new Phaser.Math.Vector2(0, 0);  
    public isRunning: boolean \= false;  
    public noiseRadius: number \= 0;

    constructor(scene: Phaser.Scene) {  
        this.scene \= scene;  
        this.setupInput();  
    }

    private setupInput() {  
        // Ловимо перший дотик у лівій частині екрана  
        this.scene.input.on('pointerdown', (pointer: Phaser.Input.Pointer) \=\> {  
            if (pointer.x \< this.scene.scale.width / 2\) {  
                this.startPoint \= new Phaser.Math.Vector2(pointer.x, pointer.y);  
                this.currentPoint \= new Phaser.Math.Vector2(pointer.x, pointer.y);  
            }  
        });

        // Рух пальця  
        this.scene.input.on('pointermove', (pointer: Phaser.Input.Pointer) \=\> {  
            if (this.startPoint && pointer.isDown) {  
                this.currentPoint.set(pointer.x, pointer.y);  
                this.calculateMovement();  
            }  
        });

        // Палець відпустили — персонаж зупиняється  
        this.scene.input.on('pointerup', () \=\> {  
            this.startPoint \= null;  
            this.currentPoint \= null;  
            this.moveVector.set(0, 0);  
            this.isRunning \= false;  
            this.noiseRadius \= 0;  
        });  
    }

    private calculateMovement() {  
        if (\!this.startPoint || \!this.currentPoint) return;

        // Рахуємо дистанцію від точки першого дотику  
        let distance \= Phaser.Math.Distance.BetweenPoints(this.startPoint, this.currentPoint);  
        let angle \= Phaser.Math.Angle.BetweenPoints(this.startPoint, this.currentPoint);

        // Обмежуємо максимальний хід джойстика  
        if (distance \> this.maxDistance) {  
            distance \= this.maxDistance;  
        }

        // Розрахунок сили нахилу (від 0.0 до 1.0)  
        let force \= distance / this.maxDistance;

        // Записуємо вектор напрямку, враховуючи силу  
        this.moveVector.setToPolar(angle, force);

        // МЕХАНІКА ШУМУ ТА СТЕЛСУ  
        if (force \> 0.5) {  
            this.isRunning \= true;  
            this.noiseRadius \= 5; // Біг: чують за 5 тайлів  
        } else if (force \> 0.1) {  
            this.isRunning \= false;  
            this.noiseRadius \= 1; // Стелс: чують тільки впритул (1 тайл)  
        } else {  
            this.noiseRadius \= 0;  
        }  
    }  
}

## **1\. Код для Кільця-Паразита: Перехоплення керування**

Це кільце втручається в роботу джойстика. Воно не просто міняє «ліво» на «право», воно має власну масу та динаміку, створюючи відчуття, що тебе дійсно «котять».

export class RingOfInvertedSubjectivity {  
    private isActive: boolean \= false;  
    private pulseTimer: number \= 0;  
    private currentPulseForce: number \= 0;

    constructor() {}

    public equip() {  
        this.isActive \= true;  
    }

    /\*\*  
     \* Перехоплює та модифікує вектор руху від джойстика  
     \* @param originalVector Вектор, який згенерував гравець пальцем  
     \* @param time Поточний час гри для розрахунку пульсації  
     \*/  
    public modifyMovement(originalVector: Phaser.Math.Vector2, time: number): Phaser.Math.Vector2 {  
        if (\!this.isActive || originalVector.length() \=== 0\) {  
            return originalVector;  
        }

        // 1\. Інверсія: розвертаємо вектор на 180 градусів (джойстик навпаки)  
        let modifiedVector \= originalVector.clone().negate();

        // 2\. Механіка пульсації (Кільце котить силою кожні 2 секунди)  
        // Синусоїда створює плавний ефект "наростання" поштовху  
        this.pulseTimer \+= 0.05;   
        this.currentPulseForce \= (Math.sin(this.pulseTimer) \+ 1\) \* 0.4; // значення від 0 до 0.8

        // Додаємо постійну тягу кільця, навіть якщо гравець ледь торкається екрана  
        if (modifiedVector.length() \< this.currentPulseForce) {  
            // Якщо гравець розгубився, кільце просто тягне його в інверсованому напрямку останнього руху  
            modifiedVector.setLength(this.currentPulseForce);  
        }

        return modifiedVector;  
    }  
}
